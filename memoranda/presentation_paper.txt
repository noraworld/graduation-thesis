【1: タイトル】
発表を始めます。タイトルは、「SNS の Cookie を利用したサイバー犯罪の犯人特定手法」です。

【2: 研究背景】
研究背景として、インターネットを利用したサイバー犯罪の被害が拡大しています。犯罪の例として、インターネット掲示板での犯罪予告や誹謗中傷、インターネット詐欺、これはたとえば、フリーマーケットサービスやオークションサービスなどで、他の出品者が出品した商品を買ったら偽物が入っていたなどです。それから、児童ポルノ画像をアップロードしたり、ダウンロードしたものを所持したりするなどの、児童ポルノ規制法に違反する行為などです。これらのサイバー犯罪の被害の拡大を防ぐために、犯人を特定して逮捕することが重要です。犯人の特定には IP アドレスが使用されますが、技術の進歩により特定が困難です。その技術の例として、キャリアグレード NAT や Tor などがあげられます。キャリアグレード NAT とは、インターネットサービスプロバイダなどの電気通信事業者が、NAT、つまりプライベート IP アドレスとグローバル IP アドレスを変換する仕組みのことです。それから、Tor とは、複数の国のサーバを経由して追跡されにくくする技術のことです。これらの技術を使用すると IP アドレスでの特定が困難となるので、IP アドレス以外の特定手法が必要になります。

【3: 提案手法 (1/2)】
そこで、本研究では、Cookie がユーザの追跡性に有効であることに着目しました。Cookie とは、Web サーバが Web ブラウザに対して任意の文字列を記憶させる仕組みです。ログイン状態を管理するときなどに使用されます。サーバがログイン状態を管理する ID などをブラウザに記憶させるように指示しておくと、それ以降、ブラウザはその ID を Cookie として同じサーバに送信するようになります。つまり、同じ Cookie を送ってきたブラウザは以前に訪問してきたユーザであることがわかります。それから、SNS にシェアボタンに着目しました。着目した理由は、最近の Web サイトには SNS のシェアボタンが設置されていることが多いからです。この 2 つに着目して、SNS のシェアボタンの Cookie から犯人の SNS アカウントを特定する手法を提案します。SNS アカウントを特定できれば、犯人を特定できる可能性が高いと考えられます。

【4: 提案手法 (2/2)】
具体的な提案手法としては、Web サイト A があり、その Web サイト A には SNS のシェアボタンが設置されています。そして、ユーザが Web サイト A に訪問します。Web サイト A に訪問すると、Web サイト A に設置されている SNS のシェアボタンを読み込むために、ユーザは SNS のサーバにもアクセスします。このとき、もしユーザがその SNS にログインしていれば、ログイン済みの Cookie を SNS のサーバに送信します。送信された Cookie を解析して、Web サイト A にアクセスしたユーザの SNS のアカウントを特定する、という提案手法です。

【5: 実験内容 (1/2)】
この提案手法の有用性を検証するために実験を行いました。実験には、SNS シェアボタンが設置された掲示板と、SNS サイトを用意しました。それから、研究室から被験者として 15 名に協力してもらいました。被験者には、まず SNS のアカウントを作成してもらいました。そして作成したアカウントにログインしてもらい、SNS にログインした状態で、掲示板にアクセスしてもらい、書込みを行ってもらいました。その後、掲示板に書込みを行った被験者の SNS のアカウントを特定する、という実験を行いました。

【6: 実験内容 (2/2)】
具体的な実験内容としては、まず、SNS にログイン済みの被験者が掲示板にアクセスして、書込みを行います。掲示板にアクセスしたときに、掲示板に設置されている SNS のシェアボタンを読み込むために、SNS のサーバに対してアクセスします。このとき、被験者は SNS にログインしている状態なので、ログイン済みの Cookie が送信されます。送信された Cookie を SNS のサーバ側で解析してアカウントを特定します。最後に、書込みが行われた時刻と Cookie が送信された時刻を照らし合わせて、この書込みを行ったユーザの SNS のアカウントはこれだというふうに特定するという実験を行いました。

【7: 実験結果】
実験結果です。被験者 15 名中、13 名を特定することができました。特定した SNS のアカウントの持ち主が本当にその書込みを行ったかどうかを被験者ひとりひとりに確認しました。その結果、すべての特定したアカウントが正しいことがわかりました。つまり、正答率は 100% でした。特定できなかった 2 名の被験者は、Cookie が送信されていなかったので、特定することができませんでした。

【8: 考察】
なぜ、Cookie が送信されなかったかというと、サードパーティ Cookie がブロックされていたからです。サードパーティ Cookie というのは、ユーザが訪問する Web サイトとは異なる Web サイトに送信される Cookie のことです。この図で説明すると、Web サイト A に訪問したユーザが、Web サイト A のサーバに対して送信する Cookie をファーストパーティ Cookie と呼びます。それに対して、Web サイト A に設置されている SNS のシェアボタンを読み込むために SNS のサーバに対して送信する Cookie をサードパーティ Cookie と呼びます。つまり、訪問する Web サイトとは違う Web サイトのサーバに送信する Cookie のことをサードパーティ Cookie と呼びます。ファーストパーティ Cookie は、ユーザが訪問する Web サイトに対して送信される Cookie なので、ユーザが送信していることを意識しやすいのですが、サードパーティ Cookie は、ユーザが送信していることを意識しにくいので、プライバシーの観点から、このサードパーティ Cookie がブロックされることがあります。今回の実験で特定することができなかった 2 名の被験者に確認したところ、このサードパーティ Cookie がブロックされるようにブラウザで設定されていたために、今回の提案手法では特定することができませんでした。

【9: まとめ】
まとめです。今回の提案手法では IP アドレスを使用していないので、IP アドレスによる特定が困難な場合でもこの提案手法は有効です。ただし、サードパーティ Cookie がブロックされると特定が困難であるという弱点があることがわかりました。以上で発表を終わります。
